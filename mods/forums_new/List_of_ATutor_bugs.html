ATutor proper

Session control

  • URL’s look RESTful but are not – links from emails often don’t work, pasted links don’t work
  • Random logouts at unpredictable time intervals, including unexplained immediate logouts
    • The amount of time elapsed that causes timeout does not seem to be constant; sometimes it can be hours, sometimes it is just minutes
    • In the worst cases (e.g., around 16:10 on 2013/01/23, inside the OCAD Learning Zone), the user is immediately logged out right after logging in

Email notifications

  • Links from email often don’t work → see Session control above
  • Email content has readability issues – Very hard to figure out which forum and which thread the notification is about

News feed (My Start Page)

Accessibility issues

  • Truncated news items confuse screen readers

Mediawiki integration

  • News feed on ATutor home page does not distinguish between the Main namespace and other namespaces (notably the User and Template namespaces), causing 404 errors when the user clicks on a news item about a page that is not in the Main namespace
  • News feed on ATutor home page does not distinguish between articles and their Talk pages, causing 404 errors when the user clicks on a news item about a Talk page
  • Apostrophes cause problems. For example, “List of links to everyone's sandboxes on Wikipedia” becomes “List of links to everyone/'s sandboxes on Wikipedia”

Forum integration

  • If a new event occurs in a shared forum, the link may point to the shared forum through a course the student is not registered in, resulting in a permission denied error when the student clicks on the link

Forums

Usability problems

  • You can only subscribe to a subforum, not a top-level forum
  • Threads can only sorted by ASCII sorting order; it is not possible to display the list of threads in a different sorting order
  • There is no intelligence in the sort – “Week 10” appears before “Week 2”
  • Current incarnation of forum session control is not tab browsing friendly [NEW since some time in Fall 2012]

Data loss

  • Subscriptions can be lost over time for no apparent reason
  • Max length attribute in post titles exceed actual field length in database, causing titles to be truncated even if the user took great lengths to create a title that fits in the edit field

Off-by-one errors

  • If a thread has 10n + 1 replies (for some n ∈ 𝐙⁺), on the thread page the forum module will display a link for the (n + 9)/10’th page, but that page will not actually contain the 10n + 1’st reply. (It will be essentially blank.)
  • If a thread has 10n + 1 replies (for some n ∈ 𝐙⁺), on the forum’s thread listing page the forum module will correctly display a functional link for the (n + 9)/10’th page, but the number of comments will be incorrectly listed as n – 1

Course content module

Accessibility issues

  • Truncated content navigation items confuse screen readers

Usability issues

  • Use of JQuery automatic timeouts causes course content to be unusable for offline use (e.g., log on to the course, realized that it’s time to go, open laptop on train, and found course content timed out and erased from the screen) [NEW since December 2012]

Profile editor

Network profile

  • The Interests and Associations fields cannot be edited (in the normal sense of the word “edit”); editing implicitly erases the old data
  • The meaning of the “Representative” section is unclear
  • The meaning of the “Name” field in the “Alternate contact” section is unclear
  • The display of the Degree column in the “Education” section feels buggy, in particular trailing slashes can occur

Assignment dropbox

  • If you pressed Delete when no file is selected, the system will still ask you “Are you sure you want to delete the following file(s)?”

MDID theme

Accessibility issues

  • Use of ° (degree sign) as bullet (in Forum Posts sidebar) confuses screen readers
